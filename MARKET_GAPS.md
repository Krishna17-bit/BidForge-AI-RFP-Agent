# Market Gap Notes for BidForge AI

## Existing market pattern
Established proposal platforms focus on enterprise response management: answer libraries, SME collaboration, review workflows, portal automation, and governance. This is valuable, but many tools are heavy for freelancers, small agencies, early-stage consultancies, and demo-led AI service sellers.

## Gap this project targets
BidForge AI is intentionally positioned as a local-first, portfolio-friendly bid intelligence agent:

1. **Small-team / freelancer fit**
   - No enterprise setup required.
   - Works from a single RFP, tender, Upwork post, or pasted client brief.
   - Useful before a sales call, not only after a formal RFP process starts.

2. **Evidence-traced bid decisions**
   - Extracts requirements.
   - Maps evidence chunks to each requirement.
   - Marks unknowns instead of pretending everything is compliant.

3. **Bid strategy, not only writing**
   - Bid/no-bid score.
   - Risks and assumptions.
   - Clarifying questions.
   - Solution architecture.
   - Delivery plan and pricing assumptions.

4. **NDA-safe public demo**
   - Uses sample data.
   - Keeps API keys in `.env`.
   - Hides provider/model controls from the UI.
   - Does not include private client projects.

5. **Portfolio advantage**
   - Shows RAG, multi-agent orchestration, retrieval/evidence mapping, proposal automation, and polished product UI in one deployable demo.
